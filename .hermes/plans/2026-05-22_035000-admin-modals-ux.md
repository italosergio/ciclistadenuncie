### Task 2: Modal de Usuários (admin-usuarios.tsx)

**Objetivo:** Substituir coluna "Ações" e confirmações inline por um modal que abre ao clicar no username.

**Arquivo:** `app/routes/admin-usuarios.tsx`

**Mudanças principais:**
1. **Remover coluna "Ações"** da tabela (th + td)
2. **Linha da tabela fica clicável** — `onClick={() => setModalUsuario(usuario)}`
3. **Novo estado:** `modalUsuario: any | null`
4. **Modal com fundo fosco** (`fixed inset-0 z-50`):
   - `bg-black/60 backdrop-blur-sm`
   - Card central: `bg-slate-900/80 border border-white/10 rounded-2xl shadow-2xl`
5. **Conteúdo do modal:**
   - Nome do usuário (bold)
   - Role badge (pill translúcido)
   - Status (Ativo/Banido) badge
   - Data de criação
   - Total de denúncias
   - Se banido: badge "BANIDO" + data do banimento (se disponível)
   - **Menu de 3 pontinhos (ícone `MoreVertical`)** canto superior direito:
     - **Enviar Contato** → abre textarea no modal. Ao enviar, mostra mensagem de sucesso tipo toast. Se futuro reply, mostra resposta + reaparece botão de contato.
     - **Suspender (se ativo) / Reativar (se banido)** → mostra mensagem de decisão "Usuário suspenso até [data]" ou "Usuário reativado com sucesso"
     - **Banir (se ativo)** → mostra mensagem "Usuário banido. Uma notificação será enviada." + opção de enviar contato
     - **Excluir Conta** → modal de confirmação com motivo (checkbox manter denúncias)
6. **Ações do 3 pontinhos** executam direto (com feedback visual)
7. **Lógica de negócio preservada:** handleRoleChange, handleBanirUsuario, handleDesbanirUsuario, handleExcluirUsuario intactas
8. **Manter confirmação inline apenas para exclusão** (que tem textarea de motivo)

## Riscos e Observações

- **Usuários:** A lógica de "enviar contato" e "responder contato" precisa de um nó extra no Firebase para armazenar as comunicações. Sugestão: usar `/contatos-usuarios/{uid}` com estrutura similar ao `/contatos`.
- **Iniciativas/Apoiadores:** Manter drag-and-drop no apoiadores — o clique no card abre modal, mas o drag não deve ser afetado (usar `onMouseDown` no handle de drag).
- **Denúncias:** A tabela deve continuar ordenada por data. O modal mostra detalhes completos.
- **Contatos:** O comportamento de expandir/ler mensagem deve ser preservado — o 3 pontinhos é adicional.
- **Planos:** A página é pública (não admin), mas o redesign visual segue o tema escuro do admin para consistência.
